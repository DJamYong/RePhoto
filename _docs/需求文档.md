随机相册回忆APP（Flutter）需求文档
1. 项目概述
1.1 项目名称
RePhoto | 重逢

1.2 项目目标

开发一款跨平台（Android + iOS）的 Flutter 应用，从用户本地相册中每日随机抽取若干照片进行展示，并允许用户对照片进行回忆记录管理。

核心目标：

提升用户“照片回忆触发体验”
将“随机浏览照片”变为“带时间线的记忆回顾”
2. 技术栈要求
前端框架：Flutter（必须）
语言：Dart
平台：Android / iOS（单代码库）
本地存储：
sqflite
权限管理：
相册权限（Android + iOS）
3. 核心功能模块
从用户相册中随机抽取照片，用户可以对照片进行浏览、删除或者填写记录。如果下一次抽到之前填写过记录的照片则可以展开查看之前的记录。

# [已实现]照片记录功能设计
去掉之前预留的hive改用sqflite数据库。
创建两张表，一张用来存放照片浏览信息、另一种用来存放照片记录
```浏览信息 (弃用，photo_history替代)
photo_views{
    photoId    照片id
    viewCount    浏览次数
    firstViewedAt   第一次浏览时间
    lastViewedAt    最后一次浏览时间
}
```
```照片记录
CREATE TABLE records (
id        主键
photoId    照片id
content     记录内容
mood        情绪  
color       标签颜色
createdAt   创建时间
updatedAt   更新时间
)
```
```浏览记录
新增 view_history 表：
列	类型	说明
id	INTEGER PK AUTO	主键
photo_id	TEXT NOT NULL	照片 ID
viewed_at	TEXT NOT NULL	浏览时间戳
```


# [已实现]抽取照片权重优化
统计近一个月照片浏览量，根据最大浏览量配置权重分比，然后再根据权重按轮盘赌抽取照片。

# 记录的情绪标签
记录文字的同时选择一个情绪（开心/平静/怀念/遗憾/温暖……），后续可以做情绪维度的回顾（"过去一年你最多的表情是怀念"），也丰富了数据维度

# 一句话卡片分享
照片 + 你的记录文字 + 照片年龄 → 生成一张精致的分享卡片（图片格式），用户可以发到朋友圈或发给朋友。这是天然的传播渠道，而且制作成本不高

# 每月/每年的回忆合集
每月 1 号自动生成上月回顾，类似 Spotify Wrapped 的逻辑。把散落的单张照片体验变成有仪式感的节点事件，也能激活沉默用户
