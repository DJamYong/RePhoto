### 5.3 照片加载流程

**权重抽选逻辑** — 近30天看得越少权重越高：

```
weight = max(1, 10 - recentViewCount)

0次 → 10  1次 → 9  2次 → 8  ...  9+次 → 1
↑ 优先展示                      ↑ 仍有基本概率
```

**`PhotoProvider`** (`photo_provider.dart:57-136`)

```
首次 build()
  └─ _loadRandomPhoto()
       ├─ PhotoService.requestPermission()
       ├─ ViewHistoryService.getRecentViewCounts()
       │    SELECT photo_id, COUNT(*) FROM view_history
       │    WHERE viewed_at >= 30天前 GROUP BY photo_id
       ├─ PhotoService.getWeightedRandomPhoto(recentViews)
       │    ├─ getAssetPathList → 全部照片元数据
       │    ├─ 每张计算 weight = max(1, 10 - count)
       │    ├─ 加权随机选择
       │    └─ return AssetEntity
       ├─ ViewHistoryService.create(photoId, now)  ← 记录这次浏览
       └─ return RandomPhotoState(photo)

refresh()  (点击"换一张")
  ├─ _loadRandomPhoto() → state = AsyncData(photo)  ← 立即显示
  └─ 后台预加载:
       ├─ Future.wait([thumbnailDataWithSize, photo.file])
       ├─ readExifFromFile(file)
       └─ state = state.copyWith(preloadedThumbnail, preloadedFile, preloadedExif)
```

## 七、预加载机制

**目的**：详情面板打开时数据已就绪，无感展示。

**触发时机**：`PhotoProvider.refresh()` (`photo_provider.dart:82-109`)

```
换一张 → state(photo)     ← 立即显示照片
                ↓
        Future.wait([
          thumbnailDataWithSize(320×320),
          photo.file
        ])
                ↓
        readExifFromFile(file)
                ↓
        state(preloadedThumbnail, preloadedFile, preloadedExif)
```

**消费组件**：

| 组件 | 预加载数据 | 回退策略 |
|---|---|---|
| `_PanelPhotoWidget` | `preloadedThumbnail` | 无预加载时 `thumbnailDataWithSize` |
| `_FileSizeTile` | `preloadedFile` | 无预加载时 `FutureBuilder(photo.file)` |
| `_ExifTile` | `preloadedExif` 或 `preloadedFile` | 无预加载时 `_loadExif()` |

## 九、所有 UI 状态

| 状态 | 组件 | 触发条件 |
|---|---|---|
| 应用加载中 | `_WarmLoading` | `photoAsync.when(loading)` |
| 权限未授予 | `_ErrorView` | `state.hasPermission == false` |
| 相册为空 | `_ErrorView` | `state.photo == null && hasPermission` |
| 加载错误 | `_ErrorView` | `photoAsync.when(error)` |
| 照片就绪 | `_PhotoAlbumView` | `state.photo != null` |
| 缩略图加载中 | `CircularProgressIndicator` | `_PhotoWidget._isLoading` |
| 缩略图失败 | `broken_image` 图标 | `_PhotoWidget._hasError` |
| 详情面板加载 | `CircularProgressIndicator` | `photoAsync.when(loading)` |
| 详情面板空 | "暂无照片" | `state.photo == null` |
| 详情面板错误 | "加载失败" | `photoAsync.when(error)` |
| 记录列表空 | 无 📋 按钮 | `records.isEmpty` |
| 记录列表有数据 | 📋 按钮 + 列表 | `records.isNotEmpty` |

## 十、关键交互流程

### 换一张

```
点击[换一张]
  PhotoProvider.refresh()
    ├─ _loadRandomPhoto()
    │    ├─ ViewHistoryService.getRecentViewCounts()  ← 近30天次数
    │    ├─ PhotoService.getWeightedRandomPhoto()
    │    │    ├─ 全部照片 × weight = max(1, 10 - count)
    │    │    └─ 加权随机选择
    │    ├─ ViewHistoryService.create()  ← 记录本次浏览
    │    └─ state(photo)  ← 立即更新 UI
    ├─ 后台: preload thumbnail + file + EXIF
    └─ state(预加载数据)
  UI: 无 AsyncLoading，照片直接替换，无转圈
```

### 删除照片

```
点击 🗑️
  showDialog 确认
    ├─ 取消 → 关闭
    └─ 删除 → PhotoService.deletePhoto(photo)
         └─ PhotoManager.editor.deleteWithIds([photo.id])
              └─ 系统弹窗二次确认（Android 10+）
         └─ 成功 → refresh()
```

### 新增记录

```
点击 ✏️
  _showRecordDialog()
    ├─ 填写内容/心情/颜色
    ├─ 保存 → RecordService.create(record)
    ├─ _recordRefreshKey++ → 📋 按钮出现
    └─ SnackBar "记录已保存"
```

### 编辑/删除记录

```
长按条目
  _showPopupMenu() → OverlayEntry 气泡菜单
    ├─ 编辑
    │   ├─ _showRecordDialog(editRecord: record)
    │   ├─ 修改 → RecordService.update(record)
    │   ├─ re-fetch → setDialogState 刷新列表
    │   └─ _recordRefreshKey++
    └─ 删除
        ├─ _confirmDelete() → showDialog 确认
        ├─ RecordService.delete(record.id!)
        ├─ setDialogState 移除条目
        └─ _recordRefreshKey++
```
