### 5.3 照片加载流程

**权重抽选逻辑** — 近30天看得越少权重越高，基准自动适配：

```
weightBase = maxCount + 1  (maxCount = 近30天最高浏览次数)
weight = max(1, weightBase - recentViewCount)

0次 → weightBase   最多 → 1
↑ 优先展示                 ↑ 仍有基本概率
```

**`PhotoProvider`** (`photo_provider.dart:57-136`)

```
首次 build()
  └─ _loadRandomPhoto()
       ├─ PhotoService.requestPermission()
       ├─ ViewHistoryService.getRecentViewCounts()
       │    SELECT photo_id, COUNT(*) FROM view_history
       │    WHERE viewed_at >= 30天前 GROUP BY photo_id
       ├─ PhotoService.getWeightedRandomPhoto(recentViews)
       │    ├─ getAssetPathList → 全部照片元数据
       │    ├─ 从往年同月同日照片中随机挑 1 张 → weight ×5
       │    ├─ 每张计算 weight = max(1, weightBase - count)
       │    ├─ 加权随机选择
       │    └─ return AssetEntity
       ├─ ViewHistoryService.create(photoId, now)  ← 记录这次浏览
       ├─ 【时间对撞检测】findSameDayPhotos(photo)
       │    ├─ 遍历全量缓存：同月同日 + 所有年份 → groups
       │    ├─ groups.length >= 2 → Random() < probability?
       │    └─ 命中 → collision = TimeCollision(groups, newestYear)
       └─ return RandomPhotoState(photo, collision?, ...)

refresh()  (点击"换一张")
  ├─ _loadRandomPhoto() → state = AsyncData(photo)  ← 立即显示
  └─ _preloadCurrentPhoto(photo)  ← 统一预加载入口
       ├─ thumbnailDataWithSize(320)      ── 独立 try/catch
       ├─ photo.file                      ── 独立 try/catch
       ├─ readExifFromFile(file)          ── 独立 try/catch
       ├─ 每步后检查 _generation 防竞态
       ├─ 最终检查 photo.id 匹配（防与删除/切换竞态）
       └─ state = state.copyWith(preloadedThumbnail, preloadedFile, preloadedExif)
```

## 八、时间对撞 (Time Collision)

### 8.1 数据模型 — `lib/models/time_collision.dart`

| 字段 | 类型 | 说明 |
|---|---|---|
| `groups` | `Map<int, List<AssetEntity>>` | 按年份分组的同月同日照片 |
| `selectedYear` | `int` | 当前浏览的年份 |
| `currentPhoto` | `AssetEntity` | groups[selectedYear].first |
| `years` | `List<int>` | 排序后的年份列表 |
| `isTriggered` | `bool` | groups.length >= 2 |

### 8.2 触发机制

**偏好配置** (`lib/providers/preferences_provider.dart:57-104`)：

| 档位 | 概率 | 说明 |
|------|------|------|
| 低 | 10% | 有匹配时 10% 展示 |
| 中 | 30% | 默认值 |
| 高 | 50% | |
| 100% | 100% | 有匹配必定展示 |

**触发流程** (`photo_provider.dart:114-128`)：

```
_loadRandomPhoto()
  └─ collisionPrefs.enabled == true
       └─ findSameDayPhotos(photo)           ← 扫描全量缓存（纯内存遍历）
            ├─ groups.length < 2  → collision = null（无跨年匹配）
            └─ groups.length >= 2 → Random() < probability?
                 ├─ 否 → collision = null
                 └─ 是 → collision = TimeCollision(groups, selectedYear: 最新年份)
```

### 8.3 UI 组件

**`_CollisionCard`** (`photo_card.dart:348-627`)

当 `state.collision != null` 时，替代拍立得卡片显示：

```
┌─────────────────────┐
│                     │
│    PageView 滑动     │  ← _YearPhotoStack（当年照片左右翻页）
│    (折叠式分页圆点)   │
│                     │
├─────────────────────┤
│ [2020][2021][2023]  │  ← 年份标签（Wrap 横向排列）
│ 2023.03.15 · 共3张  │  ← 当前照片日期 + 当年照片总数
└─────────────────────┘
```

**`_YearPhotoStack`** (`photo_card.dart:370-527`)：
- `PageView` 按年份照片列表分页
- 折叠式分页圆点指示器：≤7 张显示全部，>7 张显示 `● … ● ● ● … ●`
- 滑动 → `selectCollisionPhoto(i)` → 更新 photo + 触发预加载

### 8.4 年份与照片切换

| 方法 | 触发 | 行为 |
|------|------|------|
| `selectCollisionYear(y)` | 点击年份标签 | 切换到该年第一张，photoIndex=0，同步更新 collision.selectedYear |
| `selectCollisionPhoto(i)` | PageView 滑动 | 更新 currentPhotoIndex + photo，**后台预加载新照片详情** |
| `deleteCurrentPhoto()` | 点击删除 | 从 groups[year] 移除 → 自动选下一张 → 更新计数 → **预加载新照片** |

### 8.5 预加载覆盖

四个入口全部触发 `_preloadCurrentPhoto(photo)` (`photo_provider.dart:205-240`)：

| 入口 | 场景 |
|------|------|
| `build()` (首张) | `Future(() => _preloadCurrentPhoto)` 延迟到 state 设置后 |
| `refresh()` | 换一张 |
| `selectCollisionPhoto()` | 滑动切照片 |
| `selectCollisionYear()` | 切年份 |
| `deleteCurrentPhoto()` | 删除后自动切下一张 |

### 8.6 详情面板数据流

```
_SlidingPanel(photoAsync: stateAsync)
  └─ _PanelBody(key: ValueKey('panel_${photo.id}'))
       └─ _DrawerContent(photo, preloadedThumbnail, preloadedFile, preloadedExif)
```

`_PanelBody` 的 key 绑定到 `photo.id`，照片切换时 key 变化 → 面板销毁重建 → 所有详情更新。

## 九、数据备份与恢复

### 9.1 BackupService — `lib/services/backup_service.dart`

| 方法 | 职责 |
|------|------|
| `exportToJson()` | 查 records + view_history + prefs → JSON 字符串 |
| `validate(json)` | 校验 version、必填字段 → `ValidationResult` |
| `importFromJson(json, mode)` | 事务写入 DB + prefs，合并跳过重复 / 覆盖清空 |

**导出格式** (`version: 1`)：

```json
{
  "version": 1,
  "exported_at": "2025-07-11T10:30:00Z",
  "records": [ { "id":1, "photo_id":"abc", ... } ],
  "view_history": [ { "id":1, "photo_id":"abc", ... } ],
  "preferences": { "collision_enabled": true, "collision_probability": 0.3 }
}
```

### 9.2 UI 入口

设置页 →「数据备份」分区（始终可见，在「关于」上方）：

```
┌──────────────────────────┐
│     数据备份    ☁️        │
│  ┌──────────┐┌──────────┐│
│  │ 📤 导出  ││ 📥 导入  ││
│  └──────────┘└──────────┘│
└──────────────────────────┘
```

- **导出**：生成 JSON → `share_plus` 调系统分享面板（微信/文件/邮件）
- **导入**：`file_picker` 选 JSON 文件 → 校验 → 弹窗选合并/覆盖 → 执行

### 9.3 导入保护

- JSON 格式损坏 → 弹窗"无效的备份文件"→ 不写入
- version 不兼容 → 提示升级 App
- 合并模式：`photo_id + created_at` 判重，已有则跳过
- 全量事务：失败自动回滚

## 十、回顾数据缓存

### 10.1 ReviewCacheService — `lib/services/review_cache_service.dart`

过往月份聚合数据不可变，缓存到 `SharedPreferences` 避免重复查询。

| 缓存类型 | key 格式 | 数据 |
|---------|--------|------|
| 月总览 | `rc_ov_{year}_{month}` | `{viewedPhotos, recordedPhotos, totalChars}` |
| 每日翻看 | `rc_dv_{year}_{month}` | `[{k: day, v: count}]` |
| 记录天数 | `rc_rd_{year}_{month}` | `[1,3,15,...]` |
| 情绪分布 | `rc_md_{year}_{month}` | `[{k: "😊 开心", v: 5}]` |
| 精选照片 | `rc_tp_{year}_{month}` | `[{k: photoId, v: count}]` |
| 全部记录 | `rc_rc_{year}_{month}` | `[{id, photo_id, content, ...}]` |

**缓存策略**：
- `isPastMonth(year, month)` → true → 可缓存
- 当前月不缓存（数据仍在变化）
- 命中的查询 0 次 DB 访问

### 10.2 效果

| 场景 | 无缓存 | 有缓存 |
|------|--------|--------|
| 首次打开 2024 年度回顾 | ~48 次查询 | ~48 次 + 写缓存 |
| 再次打开 2024 年度回顾 | ~48 次查询 | **0 次查询** |
| 打开当月（2025.07）回顾 | N 次查询 | N 次查询（不缓存） |

## 十三、关键交互流程

### 换一张

```
点击[换一张]
  PhotoProvider.refresh()
    ├─ _loadRandomPhoto()
    │    ├─ ViewHistoryService.getRecentViewCounts()  ← 近30天次数
    │    ├─ PhotoService.getWeightedRandomPhoto()
    │    │    ├─ 从往年同月同日中随机挑 1 张 → ×5
    │    │    ├─ 全部照片 × weight = max(1, weightBase - count)
    │    │    └─ 加权随机选择 → 抽中历史照片时显示「此刻·彼时」
    │    ├─ ViewHistoryService.create()  ← 记录本次浏览
    │    └─ state(photo)  ← 立即更新 UI
    ├─ 后台: preload thumbnail + file + EXIF
    └─ state(预加载数据)
  UI: 无 AsyncLoading，照片直接替换，无转圈
```

### 删除照片

```
点击 🗑️
  showDialog 确认
    ├─ 取消 → 关闭
    └─ 删除 → PhotoService.deletePhoto(photo)
         └─ PhotoManager.editor.deleteWithIds([photo.id])
              └─ 系统弹窗二次确认（Android 10+）
         └─ 成功 → refresh()
```

### 新增记录

```
点击 ✏️
  _showRecordDialog()
    ├─ 填写内容/心情/颜色
    ├─ 保存 → RecordService.create(record)
    ├─ _recordRefreshKey++ → 📋 按钮出现
    └─ SnackBar "记录已保存"
```

### 编辑/删除记录

```
长按条目
  _showPopupMenu() → OverlayEntry 气泡菜单
    ├─ 编辑
    │   ├─ _showRecordDialog(editRecord: record)
    │   ├─ 修改 → RecordService.update(record)
    │   ├─ re-fetch → setDialogState 刷新列表
    │   └─ _recordRefreshKey++
    └─ 删除
        ├─ _confirmDelete() → showDialog 确认
        ├─ RecordService.delete(record.id!)
        ├─ setDialogState 移除条目
        └─ _recordRefreshKey++
```
